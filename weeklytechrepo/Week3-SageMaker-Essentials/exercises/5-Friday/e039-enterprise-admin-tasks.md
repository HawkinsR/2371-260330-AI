# Lab: Enterprise Admin Tasks (Bedrock & Costs)

## The Scenario
Your custom models are running, but now your organization wants to leverage **AWS Bedrock** for rapid prototyping of LLM features. Furthermore, the Finance department needs a clear report on AI spend. Your task is to use the **Boto3 SDK** to invoke a Bedrock Foundation Model and then simulate a **Cost Explorer** query to track the expenses associated with your project's tags.

## Core Tasks

1. **Invoke Bedrock Foundation Models:**
   - Open `e039-enterprise-admin-tasks.py`.
   - Implement the `invoke_bedrock_model` function:
     - Use `boto3.client('bedrock-runtime')`.
     - Call `invoke_model` with `modelId="anthropic.claude-3-haiku-20240307-v1:0"`.
     - Pass a JSON payload with `max_tokens`, `temperature`, and a user prompt.
2. **Handle Throttling and Quotas:**
   - Add a `try/except` block to catch `ThrottlingException` and simulate an exponential backoff retry.
3. **Analyze Project Costs:**
   - Implement the `get_project_spend` function using `boto3.client('ce')` (Cost Explorer).
   - Filter the results by the tag `Project: Project-Secure-AI`.
   - Print the total unblended cost for the current month.
4. **CloudWatch Monitoring:**
   - Simulate a CloudWatch alarm that triggers if `ModelLatency` exceeds 500ms.

## Definition of Done
- A functional Python script that invokes a Bedrock model via Boto3.
- Successful parsing and display of LLM responses and token usage statistics.
- A cost report generated by filtering AWS resources by project tags.
- Documented handling of API throttling and latency monitoring.
