# Lab: Managing Conversation History

## The Scenario
Your company's customer support bot keeps crashing because users chat with it for hours, exceeding the LLM's context window. Additionally, an external security audit found a vulnerability: the LLM is being trusted to provide the `user_id` when calling the secure `delete_account` tool. You must implement a middleware message trimmer to manage conversation history and utilize LangGraph's Configuration tools to strictly inject the authenticated `user_id` securely at runtime.

## Core Tasks

1. Navigate to the `starter_code/` directory.
2. Open `e047-e047-state_management_lab.py`.
3. Complete the `trim_chat_history` function:
   - This function acts as middleware before the LLM generates a response.
   - You are provided a large `chat_history` list.
   - Always retain the System Prompt (the first message at index 0).
   - Calculate how many messages to retain from the end based on `max_history`.
   - Extract the last `max_history` messages.
   - Combine the System Prompt and the retained messages into a new list and return it.
4. Complete the `secure_delete_tool` function:
   - This represents a secure endpoint. It receives an `account_name` provided by the LLM, but the `injected_user_id` must come from our secure global state configuration.
   - Check if the `injected_user_id` strictly equals `"AUTH_USER_999"`.
   - If yes, return `"Account securely deleted."`
   - If no, return `"ERROR: Unauthorized."`
5. Complete the `execute_secure_tool_call` function:
   - Extract the authenticated user ID from the nested `graph_config` dictionary (it's located at `["configurable"]["authenticated_user_id"]`).
   - Find the tool parameters generated by the LLM in `llm_generated_args`.
   - Call the `secure_delete_tool` passing the LLM arg and the injected secure ID.
   - Print the tool's return value.

## Definition of Done
- The script executes successfully.
- The chat history is accurately trimmed to exactly 3 messages (System Prompt + the last 2 interactions).
- The simulated tool rejects a fake user ID but correctly authorizes the simulated run utilizing the injected state.
