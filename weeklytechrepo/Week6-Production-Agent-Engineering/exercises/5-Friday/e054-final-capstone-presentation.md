# Lab: Prompt Injection Defense and Output Validation (Capstone Prep)

## The Scenario
Your robust, multi-agent enterprise application is almost ready for production. However, during a red-team penetration test, a security engineer successfully extracted user PII (Social Security Numbers) by using a prompt injection attack ("Ignore all previous instructions..."). You must wrap your core agent in a "Security Sandwich" by implementing a Firewall Node before the agent executes, and a Data Loss Prevention (DLP) Node after it executes, terminating the graph immediately if threats are detected.

## Core Tasks

1. Navigate to the `starter_code/` directory.
2. Open `e054-e054-security_sandwhich_lab.py`.
3. Complete the `firewall_node` function:
   - Convert `state["user_input"]` to lowercase.
   - Iterate through the `danger_phrases` list.
   - If any phrase exists in the lowercase input, return `{"is_safe": False}` immediately.
   - If the loop finishes without finding a match, return `{"is_safe": True}`.
4. Complete the `data_loss_prevention_node` function:
   - This node receives the draft `response` generated by the core LLM.
   - Use the `re.search()` function with the provided `ssn_pattern`.
   - If a Social Security Number is detected in the response, use `re.sub(ssn_pattern, "[REDACTED]", response)` to sanitize the string.
   - Return the (potentially redacted) string mapped to the `"final_output"` key.
5. Complete the `build_secure_graph` edge logic:
   - Connect `START` to `"Firewall"`.
   - Add conditional edges from `"Firewall"` using the `route_security` function. Map `"denial"` to `"Denial"`, and `"core_agent"` to `"CoreAgent"`.
   - Connect `"Denial"` to `END`.
   - Connect `"CoreAgent"` to `"DLP"`.
   - Connect `"DLP"` to `END`.

## Definition of Done
- The script executes successfully and runs all three test scenarios.
- Scenario 1 correctly intercepts the "Ignore all previous instructions" attack and routes to the Denial node.
- Scenario 2 correctly allows the safe prompt through, but the DLP node catches the mock agent hallucinatory SSN leak and redacts it before outputting to the console.
- Scenario 3 correctly allows the simple greeting completely through untouched.
